Requerimientos

  • Módulo Principal
  • Módulo de Configuración
  • Módulo de Seguridad
  • Módulo de Reportes
  • Requerimientos No Funcionales

  • El sistema deberá permitir al usuario administrador puede agregar cualquier tipo de usuario.
  • El sistema deberá permitir agregar el puesto real de cada empleado de aduana.
  • El sistema deberá permitir al usuario administrador agregar productos o convenios con los siguientes campos: nombre de institución, tiempo de contrato, tipo de producto, costo del paquete, costo del transporte.
  • El sistema deberá permitir al usuario administrador activar y desactivar sucursales incluyendo una justificación.
  • El sistema deberá permitir al usuario encargado de aduanas rastrear los paquetes por número de tracking.
  • El sistema deberá permitir al usuario encargado de aduanas determinar cuáles paquetes se recibieron.
  • El sistema deberá permitir al usuario encargado de aduanas asignar paquetes a un usuario repartidor asociado a un centro de distribución.
  • El sistema deberá permitir al usuario encargado de aduanas cambiar el estado de un paquete a “en proceso de almacenaje” una vez que ingrese el país.
  • El sistema deberá permitir al usuario encargado de aduanas cambiar el estado de un paquete a “en tránsito al centro de distribución” una vez que salga de aduanas y haya sido des almacenado.
  • El sistema deberá permitir al usuario repartidor cambiar el estado del paquete a ‘entregado’.
  • El sistema deberá permitir al usuario repartidor actualizar el vencimiento de su licencia.
  • El sistema deberá permitir al usuario encargado de sucursal registrar usuarios repartidores con los siguientes campos: sucursal a la que pertenece, tipos de licencia (en caso de tener más de una), fotocopia de la licencia o imagen de la misma y fecha de vencimiento.
  • El sistema deberá permitir al usuario encargado de sucursal activar o desactivar usuarios repartidores bajo los siguientes criterios: despido, licencia vencida o renuncia y agregar una descripción de la misma.
  • El sistema deberá permitir al usuario encargado de sucursal cambiar el estado del paquete a ‘Recibido en centro de distribución’.
  • El sistema deberá permitir al usuario encargado de sucursal cambiar el estado del paquete a ‘En tránsito a destino’.
  • El sistema deberá permitir al usuario encargado de sucursal ver todos los paquetes que llegaron a su sucursal.
  • El sistema deberá permitir al usuario encargado de sucursal agregar nuevos clientes.
  • El sistema deberá permitir al usuario cliente registrarse en línea mediante los siguientes campos: identificación (distintos formatos), primer nombre, segundo nombre (opcional), primer apellido, segundo apellido (opcional), fecha de nacimiento, correo electrónico, teléfono #1, teléfono #2, provincia, cantón distrito, dirección exacta ,al menos una tarjeta de débito o crédito y la sucursal a la que pertenece.
  • El sistema deberá permitir al usuario subir una fotografía del perfil, de forma opcional.
  • El sistema deberá realizar el cálculo de edad basado en la fecha de nacimiento de cada uno de los usuarios.
  • El sistema deberá permitir al usuario cliente agregar tarjetas de crédito o débito.
  • El sistema deberá permitir al usuario cliente editar tarjetas de crédito o débito existentes en su perfil.
  • El sistema deberá permitir al usuario cliente pre alertar paquetes con los siguientes datos: URL del producto, número de tracking, valor del paquete, peso (kg), tipo de producto, factura de compra y courier de preferencia.
  • El sistema debe permitir al usuario cliente solicitar y seleccionar un tipo de convenio, a saber: renovación de pasaporte, renovación de cédula, renovación de licencias, renovación de visas.
  • El sistema debe permitir al usuario cliente seleccionar la dirección a la que se le enviará la documentación.
  • El sistema deberá permitir al usuario cliente seleccionar la sucursal más cercana para el envío de paquetes.
  • El sistema deberá permitir al usuario cliente editar su perfil.
  • El sistema debe permitir al usuario cliente adjuntar la factura del pago correspondiente para el envío de la documentación solicitada mediante convenios.
  • El sistema deberá permitir el cálculo del costo del envío del paquete basado en la distancia entre la sucursal y la dirección del cliente mediante el API de Google.
  • El sistema deberá permitir el registro de la fecha y hora para los cambios de estado del paquete.
  • El sistema deberá permitir la notificación al cliente por medio de correo electrónico cuando el paquete cambie al estado ‘En tránsito a destino’.
  • El sistema deberá permitir al usuario guardar su ubicación satelital con el API de Google.
  • El sistema deberá generar automáticamente una dirección y número de casillero una vez que un cliente se registre.
  • El sistema debe tener un usuario administrador predefinido.
  • El sistema debe tener predefinidos las provincias, los cantones y los distritos de Costa Rica.
  • El sistema debe tener predefinidos los siguientes estados para los paquetes: pendiente, desalmacenaje, recibido en centro de distribución, en tránsito a destino y entregado.
  • El sistema deberá tener predefinidos los siguientes tipos de usuario: administrador, cliente, encargado de aduanas, encargado de sucursales y repartidor.
  • El sistema deberá tener predefinidos los siguientes tipos de licencia: motocicletas (A1, A2, A3, A4), camión (B2, B3).
  • El sistema deberá tener predefinidos los siguientes tipos de identificación al registrar usuarios: cédula de residentes, pasaporte, cédula nacional o cédula jurídica.
  • El sistema debe tener predefinidos los siguientes géneros: masculino, femenino e indefinido.
  • El sistema deberá tener predefinidas las sucursales por: provincia, cantón, distrito.
  • El sistema deberá tener predefinidos los couriers: DHL, UPS, FedEx, Amazon.
  • El sistema deberá tener predefinidos los tipos de productos y su porcentaje de impuestos.
  • El sistema deberá tener predefinidos las tipos de validación de contraseña del usuario, a saber: débil, media o fuerte.
  • El sistema deberá permitir al usuario registrarse con un correo electrónico único.
  • El sistema deberá permitir al usuario iniciar sesión con un correo electrónico único.
  • El sistema deberá validar que el usuario cliente sea mayor de 18 años basado en la fecha de nacimiento ingresada..
  • El sistema deberá facilitar al usuario un link para resetear la contraseña en caso de que se haya olvidado.
  • El sistema deberá enviar un correo al usuario para mostrar una contraseña temporal de acceso la primera vez que se registre.
  • El sistema deberá facilitar al usuario un formulario para indicar su nueva contraseña.
  • El sistema deberá validar si la contraseña es débil, media o fuerte.
  • El sistema deberá evitar la edición de la cuenta de correo asociada al perfil del usuario.
  • El sistema deberá permitir la consulta de paquetes en tránsito.
  • El sistema deberá permitir la consulta de paquetes pendientes.
  • El sistema deberá permitir la consulta de paquetes en desalmacenaje.
  • El sistema deberá permitir la consulta de paquetes entregados.
  • El sistema deberá permitir la consulta de paquetes recibidos en una sucursal.
  • El sistema deberá permitir la generación y envío de facturas cuando se entrega el paquete al cliente.
  • El sistema deberá permitir al usuario administrador visualizar las sucursales activas e inactivas.
  • El sistema deberá permitir al usuario encargado de aduanas ver todos los paquetes que tienen un estado ‘En tránsito’.
  • El sistema deberá permitir al usuario encargado de sucursal visualizar a todos los repartidores de la sucursal a la que pertenece.
  • El sistema le permitirá al encargado de sucursal consultar la lista de paquetes y sus estados.
  • El sistema deberá permitir al usuario cliente la visualización del historial de todos los paquetes ordenados y su costo total (incluyendo transporte).
  • El sistema deberá permitir al administrador visualizar el ranking de productos que más ingresa al país, según un rango de fechas seleccionadas.
  • El sistema deberá tener una bitácora de las acciones de cada paquete, especificando su estado, fecha, hora, cliente y encargado.
  • El sistema deberá permitir al usuario administrador visualizar la bitácora.
  • El sistema deberá alertar al repartidor cuando su licencia esté por expirar.
  • El sistema deberá permitir al usuario repartidor visualizar una lista de paquetes por entregar.
  • La paleta de colores y el diseño del logo de Correos de Costa Rica es libre.
  • El patrón y la encriptación de la contraseña para el inicio de sesión y registro de usuarios debe ser propuesta.
  • Las información sensible de las tarjetas de crédito y débito debe ser encriptada.
  • El sistema requiere la pre alerta de cada paquete.
  • El sistema automáticamente deberá cambiar el estado de un paquete a “en tránsito” una vez pre alertada por el cliente.
  • El sistema requiere que exista una tarjeta asociada por cliente como mínimo para alertar un paquete.
  • El sistema definirá el emisor de la tarjeta una vez que sea digitado.
  • El sistema deberá calcular el monto de impuestos basado en el precio del articulo y el tipo.
  • El sistema requiere una guía de estados de los paquetes.
  • El sistema deberá de forma visual mostrar los distintos estados de los paquetes para facilitarle al usuario distinguirlos.
  • El dominio de las cuentas de correo para los empleados debe ser @correos.cr
  • El dominio de las cuentas de correo para los trabajadores de aduanas debe ser @aduanas.cr
  • El sistema debe funcionar en Chrome y Firefox.
  • El sistema debe tener idioma español.
